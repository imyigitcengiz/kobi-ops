Sunucu / Docker kurulumu (Coolify)

WhatsApp köprüsü panel konteynerinden ayrı bir servistir (Node + Chromium). Sadece panel Dockerfile kurduysanız köprü çalışmaz; panel içinden de başlatılamaz.

  • Önerilen: Coolify kaynağını Docker Compose yapın — repo kökündeki compose.yaml (veya deploy/coolify/compose.yaml) hem app hem whatsapp-bridge açar.
  • Panel ortam değişkeni (Compose içinde otomatik; tek panel uygulamasında elle yazın): WHATSAPP_BRIDGE_URL={{ whatsapp_bridge_url_docker|default:"http://whatsapp-bridge:3939" }} DJANGO_WHATSAPP_BRIDGE_CAN_SPAWN=0
    • {% if whatsapp_bridge_url_is_local %}
  • Şu an panelde {{ whatsapp_bridge_url }} görünüyor — Docker içinde bu adres yanlış (panel kendine bakar). Coolify Environment’a yukarıdaki whatsapp-bridge URL’sini yazın veya Compose kullanın.
    • {% else %}
  • Şu an ayarlı adres: {{ whatsapp_bridge_url }} — köprü servisi ayakta ve aynı Docker ağında olmalı.
    • {% endif %}
  • İki ayrı Coolify uygulaması: köprü için deploy/whatsapp-bridge/Dockerfile, panelde WHATSAPP_BRIDGE_URL=http://<köprü-internal-host>:3939 (127.0.0.1 değil).
  • Detay: DEPLOY.md ve deploy/coolify/README.md
